# Loan Approval Prediction

A beginner-level binary classification project that predicts whether a home-loan application gets **approved** based on demographic and financial features.

## Problem Statement

Given an applicant's gender, marital status, dependents, education, employment, income, requested loan amount, term, credit history, and property area, predict whether **Dream Housing Finance** will approve the loan (`Loan_Status = Y` or `N`).

## Dataset

- **Source**: [Loan Prediction Problem Dataset (Analytics Vidhya / Kaggle)](https://www.kaggle.com/datasets/altruistdelhite04/loan-prediction-problem-dataset) — mirrored on GitHub
- **Samples**: 614 applications (422 approved, 192 denied)
- **Features**: 11 inputs + 1 binary target

| Feature | Type | Description |
|---------|------|-------------|
| Loan_ID | string | Unique application id (dropped) |
| Gender | categorical | Male / Female |
| Married | categorical | Yes / No |
| Dependents | string -> int | 0 / 1 / 2 / 3+ (normalized to integer) |
| Education | categorical | Graduate / Not Graduate |
| Self_Employed | categorical | Yes / No |
| ApplicantIncome | numeric | Applicant monthly income |
| CoapplicantIncome | numeric | Co-applicant monthly income |
| LoanAmount | numeric | Requested loan in thousands |
| Loan_Amount_Term | numeric | Term in months (typically 360) |
| Credit_History | binary | 1 = clean credit history, 0 = adverse |
| Property_Area | categorical | Urban / Semiurban / Rural |
| **Loan_Status** | **binary** | **Y = approved (1), N = denied (0)** |

7 of 12 columns have missing values; mode/median imputation is used during cleaning.

## Results

7 classification models were trained on the engineered, one-hot encoded feature space, and the best baseline was further tuned with `GridSearchCV`.

| Model | Accuracy | Precision | Recall | F1 Score |
|-------|----------|-----------|--------|----------|
| KNN (K=3) | 0.8618 | 0.8542 | 0.9647 | 0.9061 |
| SVM | 0.8537 | 0.8317 | 0.9882 | 0.9032 |
| Logistic Regression | 0.8537 | 0.8384 | 0.9765 | 0.9022 |
| Random Forest | 0.8537 | 0.8602 | 0.9412 | 0.8989 |
| Naive Bayes | 0.8455 | 0.8367 | 0.9647 | 0.8962 |
| Gradient Boosting | 0.8374 | 0.8421 | 0.9412 | 0.8889 |
| Decision Tree | 0.7967 | 0.8191 | 0.9059 | 0.8603 |
| **Random Forest (Tuned)** | **0.8537** | **0.8454** | **0.9647** | **0.9011** |

Best GridSearchCV parameters: `n_estimators=300, max_depth=5, min_samples_split=2, min_samples_leaf=1` (CV F1 = 0.8686).

## Key Findings

- **Credit_History dominates**: ~79% approval rate when `Credit_History == 1`, ~8% when `0`. Almost all model importance flows through this column.
- **Property_Area = Semiurban** has the highest approval rate (~77%) — even higher than Urban or Rural.
- **Married graduates** are the strongest demographic combination for approval.
- **Income is mildly informative**: approved applicants average slightly *lower* incomes than denied — counterintuitive at first glance, but explained by `Credit_History` correlating with approval more than income does.
- **Most models cluster around F1 ≈ 0.90** because the task is largely solvable by a simple rule on `Credit_History`. The challenge is teasing out the remaining 10–15% of edge cases.
- **6 of 12 columns have missing values**; mode (categorical) and median (numeric) imputation works well here. `Credit_History` is missing for ~50 rows and we impute with the mode (1.0) since most applicants have a clean history.
